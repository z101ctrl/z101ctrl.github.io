Server setup

Mar 18, 2018

During the COVID pandemic I found myself with quite an amount of free time, so I became really obsessed with linux systems. Aproximately one and a half year ago I got an old Lenovo into my hands, so I opted to repurpose it into a server.

In this post I’m describing the general setup of my server, that has been running with almost no downtime for ten months Firstly, I decided to put Debian as it has already well tested stable software in its repositories. As I already had mounted a server before, I realized that installing services one by one wasn’t really worth it as if there’s any problem with the machine you’d have to redo the installation process all over again.

Knowing that, I decided to use docker, as it allows to install software in its virtual machine-like environment by specyfing the services you want to install in a single file.

I also knew that I didn’t want to expose my server to the internet, so opening ports was no option for me. Instead, I opted to access via a VPN. At first I tried self-hosting a Wireguard instance, but it didn’t prevail due to my university network not letting me communicate via UDP connections. Setting up the connection via HTTP didn’t work either. Then tried Tailscale, but I discarded it for the exact same reason. Finally, after a long research, I came across ZeroTier. It not only worked flawlessly, as it also had several advantages over the previously mentioned services.

  1. For using Wireguard you need an static IP address (currently my internet provider goes through a CGNAT), which not many internet providers give you. ZeroTier doesn’t need that as it informs whatever IP address you have to the ZeroTier central server.
  2. ZeroTier uses UDP hole punching, so the university network connection problem is essentially fixed with that.
  3. As I’m using the official ZeroTier servers, I can easilly add new devices from their webpage. This being an advantage and a disadvantage at the same time, as I really wanted to self-host everything.

Having that, I proceded to install the services that I needed. Those being:

  • Nextcloud: google drive like functionality.
  • Jellyfin: media center for storing music, movies, etc.
  • Portainer: nice GUI manager for the docker images.
  • Ntfy: I use this to send push notifications to my cellphone when i have to run long processes. This + TMUX is a game changer. Also, it has nice integrations.
  • Nginx Proxy Manager: reverse proxy manager, more on this later.
  • Transmission: torrent downloader for downloading totally legal things.
  • Homarr: nice home page where I have all my services displayed.

[Foto de homarr]

Nice, but now each time I want to access my server I have to write my ugly IP address. For solving that I configured a DNS entry using DuckDNS + Let’s Encrypt for the SSL certificates, so instead of writing “186.x.x.x” I’ll have to write “genericname.duckdns.org”. Much better.

Another problem was that each time I wanted to access certain service I had to write it by specifying the port in which the service is listening. For this I configured the Nginx Proxy Manager, so if i want to access my jellyfin instance instead of writing “genericname.duckdns.org:171” I’ll have to write “jellyfin.genericname.duckdns.org”.
